# 脚本框架生成

## 前置

1. 读 `00-人设中枢/` 三份档案。为空则停下，先引导采访
2. 确定写哪个选题：
   - 他指明了就用那个
   - 没指明就列出 `02-选题库/已定/` 的选题让他挑；已定为空就列 `待评估/` 里的高分选题问他「先定一个？」
3. 问一个问题再动笔：「这条打算做成什么形式？」（口播短视频 / 图文 / 长文章。默认按 1 到 3 分钟口播短视频写）

## 框架结构（七步说服链）

按顺序生成，每一步都标注「这步的任务」：

1. **钩子（前 3 秒）**：给 3 个备选，风格必须符合语言指纹。可用套路：反常识结论直给 / 具体数字冲击 / 冲突现场感。禁止「今天我们来聊聊」式开场
2. **建立资格（为什么听我说）**：从人设档案的差异化资产里选一条，一句话带过，不自夸
3. **戳痛点**：受众正在经历的具体场景，用画面感描述，别用抽象词
4. **给认知（核心观点）**：从观点库或该选题的核心火种展开，这是全片的「一个」重点，只讲一个
5. **给证据**：标注【这里放你自己的案例：建议讲 XXX 那件事，重点讲清楚 A、B 两个细节】。案例线索从人设档案和选题来源里找，找不到就写【需要你补一个亲身案例，要求：…】
6. **预埋反驳**：受众最可能的一句质疑 + 回应思路
7. **结尾行动指令**：对应人设一页纸里「受众看完要做的动作」，一句话

## 留白规则（关键）

框架只搭骨头，血肉必须留给他：

- 核心案例、具体数字、当事人对话，一律用【留白标注】写清楚这里需要他填什么、什么标准算填好了
- 每段给出「说什么」和「怎么说的示例句」，示例句必须像他本人（对照语言指纹），并标注（示例，可改）
- 全文框架控制在 500 字以内，留白越多越好，写太满他就只会照念，一照念就假

## 产出

存入 `03-脚本框架/选题标题-框架.md`，文件头部带：选题来源链接、形式、生成日期、预估成片时长。

收尾告诉他：「框架在 03-脚本框架 里，把【留白】填了就能拍。填完想让我把把关，说『审稿』。」
