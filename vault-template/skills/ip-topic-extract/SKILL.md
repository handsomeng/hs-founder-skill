# 选题提取

## 前置检查

1. 读 `00-人设中枢/` 三份档案。为空则停下，引导他先「开始采访」
2. 读 `02-选题库/_已处理清单.md`（没有就创建一个空的），确认哪些收件箱文件已经处理过
3. `01-收件箱/` 里没有新文件时，告诉他「收件箱是空的」，并提议：「也可以现在跟我聊十分钟最近在忙什么、气什么、得意什么，我从聊天里帮你挖选题。」聊天内容按下面同样流程处理

## 提取流程

对每个未处理的收件箱文件：

1. 通读全文，找出所有可能成为内容的「火种」：一个观点、一个故事、一次冲突、一个反常识判断、一个具体数字
2. 每个火种对照人设档案问三个问题：
   - 和他的定位相关吗？（不相关的丢弃，不进库）
   - 目标受众会关心吗？
   - 他有独家资格讲这个吗？（亲身经历 / 一手数据 / 反共识立场）

## 打分（120 分制）

每个候选选题打分，写明每项得分理由：

- **人设匹配度（40 分）**：和一句话定位的贴合度、是否用上差异化资产
- **爆款潜力（60 分）**：故事性 15 + 争议性 15 + 情绪共鸣 15 + 实用价值/社交货币 15
- **时效性（±20 分）**：蹭上近期热点 +，过时话题 −

分级：≥90 优先做；70 到 89 可做；<70 不进库，一句话汇报后丢弃。

## 产出格式

每个入库选题在 `02-选题库/待评估/` 建一个文件，文件名就是选题标题：

```
# （选题标题，按他的语言风格写，别用 AI 腔）

- 总分：X / 120（各项得分一行列明）
- 来源：收件箱哪个文件的哪句话（引用原话）
- 一句话讲什么：
- 为什么是他来讲：（对应的差异化资产）
- 建议切入角度：（给 1 到 2 个）
- 提取日期：YYYY-MM-DD
```

处理完把文件名记入 `02-选题库/_已处理清单.md`。

## 收尾汇报

用一段话汇报：处理了几个文件、挖出几个选题、最高分是哪个、为什么值得先做。然后告诉他：「看得上的，跟我说『第几个定了』，我帮你挪进已定文件夹。想直接开写就说『出框架』。」

他说某个选题定了：把该文件从 `待评估/` 移到 `已定/`。他说某个不要：删除该文件（这是唯一允许删文件的场景）。
