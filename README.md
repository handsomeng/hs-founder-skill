# hs-founder-skill：创始人 IP 内容系统

一套开箱即用的 AI 内容流水线，为想做个人 IP 但不懂技术的创始人设计。核心是五个环节：**采访建档 → 选题提取 → 脚本框架 → 审稿 → 知识库管理**，全部通过和 AI 说话完成。

- 零安装依赖：skill 直接内置在知识库文件夹里，`AGENTS.md`（Codex）和 `CLAUDE.md`（Claude Code）双路由，拷走文件夹就能用
- 人机分工明确：AI 负责结构、提炼、把关；核心案例和真实经历必须由本人填写，框架刻意留白，防止内容变成 AI 腔
- 全程 git 保底：AI 每次写入自动提交，改坏了随时回滚

## 安装

前提：电脑装好 git 和 Codex CLI（或 Claude Code）。启动 AI，说一句：

> 安装 https://github.com/handsomeng/hs-founder-skill 并配置我的 Obsidian 知识库

AI 会读到下方的安装指令自动完成，最后引导你打开 Obsidian 并重启会话。

### 给 AI 的安装指令（用户说出上面那句话时照此执行）

自动完成以下全部步骤，过程中不问技术问题：

1. 问用户一个问题：「知识库文件夹想叫什么名字？（不说就默认叫 我的IP）」
2. `git clone https://github.com/handsomeng/hs-founder-skill.git /tmp/hs-founder-skill`
3. 把 `/tmp/hs-founder-skill/vault-template` 复制为 `~/<用户起的名字>`（家目录直接子目录；macOS 上禁止放 Downloads / Documents / Desktop，系统会拦 AI 读写）
4. 在新文件夹里 `git init && git add -A && git commit -m "init"`
5. 删除 `/tmp/hs-founder-skill`
6. 用大白话告诉用户剩下两个必须人工做的动作：
   - 打开 Obsidian，「打开文件夹作为仓库」，选刚建的文件夹
   - 退出当前 AI 会话，进入该文件夹重新启动（`cd ~/文件夹名` 后运行 codex 或 claude），然后说「开始采访」

## 使用：五句口令

在知识库文件夹里启动 AI 后，只需要记住五句话：

| 口令 | 干什么 |
| --- | --- |
| **开始采访** | 首次使用必做。AI 按五个板块和你开放式聊 30 分钟，生成人设档案（定位、语言风格、观点库），是后面一切的地基。每月说「更新人设」增量补充 |
| **提取选题** | 把语音转文字、会议记录、随想扔进 `01-收件箱/`，AI 对照你的人设提取选题并打分（120 分制）。收件箱空着也行，直接和 AI 聊十分钟让它现场挖 |
| **出框架** | 从选题库挑一个，AI 按七步说服链出脚本骨架，核心案例处留白等你填 |
| **审稿** | 写完的稿子给 AI，三道检查（结构 / 像不像你 / 去 AI 味），只出修改清单不代写 |
| **收纳 / 整理知识库** | 丢任何资料给 AI 自动归档建链；「帮我找」可全库问答；每周说「整理知识库」做一次体检 |

## 目录结构

```
你的知识库/
├── AGENTS.md / CLAUDE.md    AI 操作手册 + 口令路由（两文件内容相同）
├── skills/                   五个模块的详细执行手册
├── 00-人设中枢/              人设一页纸、语言指纹、观点库（采访生成）
├── 01-收件箱/                原始素材唯一入口，一天一个文件
├── 02-选题库/                待评估 / 已定
├── 03-脚本框架/              AI 出骨架，你填血肉
├── 04-成稿归档/              发布过的最终版
└── 05-数据复盘/              发布数据回填，反哺选题打分
```

每个目录有 `_说明.md`，人和 AI 都看得懂。

## 让系统活下来的一个习惯

整套系统成败在收件箱：每天把微信语音转文字、备忘录随想、会议记录粘进 `01-收件箱/当天日期.md`。喂得越勤，选题越多。建议明确到人和时间（本人或助理，每天固定时段）。

## 定制

改 skill 逻辑直接编辑 `skills/*/SKILL.md`，口令路由在 `AGENTS.md`（改完同步复制一份为 `CLAUDE.md`），改完 git commit。整套无行业假设，行业属性来自采访生成的人设档案。
